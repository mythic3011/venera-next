# M27 Source Runtime SDK and Normalization Layer

## Goal

- Turn the source JavaScript runtime from a bare script executor into a constrained source adapter SDK.
- Provide common runtime helpers for HTTP, HTML parsing, JSON validation, typed source errors, tag normalization, OpenCC conversion, diagnostics redaction, and source capability checks.
- Reduce duplicated parsing/error-handling code inside individual source scripts.
- Prevent source scripts from throwing raw JavaScript errors into app UI.
- Preserve existing source behavior through compatibility wrappers.

## Problem

Current source runtime behaves too much like a raw JavaScript engine.
It can execute source scripts, but common platform responsibilities are not clearly owned:

- HTML parsing
- API JSON response validation
- `code != 200` response handling
- missing field handling
- pagination shape validation
- chapter grouping
- tag/name normalization
- simplified/traditional Chinese conversion
- network retry / timeout policy
- auth/session redaction
- diagnostics safety
- typed error mapping
  As a result, every source script must hand-roll behavior. When API response shape changes or returns a non-success domain code, script logic can throw raw errors such as:

```text
ReferenceError: res is not defined
TypeError: cannot read property 'name' of undefined
NoSuchMethodError: Class 'String' has no instance getter 'entries'

This is not a source-platform model. It is only script execution.

Scope

* source runtime SDK helpers
* source script compatibility wrapper
* typed source result model
* typed source error model
* HTTP response validation helpers
* HTML parser helper
* JSON schema/path helper
* tag/name normalization helper
* optional OpenCC conversion helper
* diagnostics redaction helper
* source capability declaration
* plugin API versioning

Out of scope:

* no ReaderNext route semantics change
* no ReaderNext render pipeline rebuild
* no appdata migration change
* no cache migration change
* no cookie migration change
* no source repository management UI change
* no source package registry change
* no source install/update workflow rewrite
* no bypass logic for third-party source restrictions
* no network signing/auth redesign

Hard Rules

1. Source runtime must expose a versioned SDK surface.
2. Source scripts must not depend on unversioned app internals.
3. Source scripts must not hand-roll common response validation where SDK helpers exist.
4. code != 200 API responses must become typed source errors.
5. Missing required fields must not throw raw JavaScript TypeError.
6. Undefined variables inside source wrappers must become typed runtime failures with source/stage context.
7. HTML and JSON source modes must both be supported.
8. OpenCC conversion must be application-owned or SDK-owned, not duplicated across scripts.
9. Tag normalization must be deterministic.
10. Normalization must not destroy original source text.
11. Source runtime diagnostics must redact tokens, cookies, signatures, device IDs, pseudo IDs, and auth headers.
12. Source script contents must not be dumped into diagnostics by default.
13. Source runtime must return typed results to Dart/app layer.
14. Source runtime must expose visible typed UI error states, not silent empty UI.
15. Source runtime must support compatibility mode for existing scripts.
16. SDK helpers must be additive first; existing scripts must not be broken in M27-A.
17. Source capability declaration must be explicit.
18. Source scripts must not be allowed to mutate global app settings directly.
19. Runtime must separate source-specific logic from application-owned policy.
20. M27 must not change ReaderNext cutover/fallback semantics.

Runtime Responsibility Model

Application
- owns storage
- owns UI
- owns settings
- owns diagnostics policy
- owns language preference
- owns source install/update policy
- owns cache policy
Source Runtime SDK
- provides safe capabilities to scripts
- validates common response shapes

- normalizes text/tags where requested
- redacts diagnostics
- controls runtime API version
Source Script
- implements source-specific URL construction
- maps source-specific payload to common comic/chapter/page models
- declares capabilities
- avoids direct app internals

Source Runtime SDK Surface

1. HTTP Helper

runtime.http.request({
  method: "GET",
  url,
  headers,
  query,
  body,
  timeoutMs,
  responseType: "json" | "html" | "bytes" | "text",
  redact: ["authorization", "cookie", "x-auth-signature"]
})

Expected behavior:

* applies app network policy
* supports timeout
* returns typed response
* redacts sensitive headers in logs
* does not throw raw network exception unless wrapped

Typed response:

{
  ok: true,
  status: 200,
  headers: {},
  body: {},
  rawBodyPreview: "<redacted or limited>"
}

Typed failure:

{
  ok: false,
  error: {
    code: "NETWORK_REQUEST_FAILED",
    message: "Request failed",
    stage: "loadInfo"
  }
}

2. API JSON Helper

runtime.json.expectApiSuccess(response, {
  successCode: 200,
  codePath: "code",
  messagePath: "message",
  resultsPath: "results",
  stage: "loadInfo"
})

Expected behavior:

* if code == 200, returns results
* if code != 200, throws/returns typed SourceBlocked or SourceUnavailable
* preserves source message as user-safe message where allowed
* does not allow downstream code to read missing results.comic.name

Example mapping:

code=210 -> SOURCE_BLOCKED or SOURCE_REQUIRES_OFFICIAL_APP
code=401 -> SOURCE_AUTH_REQUIRED
code=403 -> SOURCE_FORBIDDEN
code=404 -> SOURCE_NOT_FOUND
network failure -> SOURCE_UNAVAILABLE
schema mismatch -> SOURCE_SCHEMA_INVALID

* v1 scripts: compatibility wrapper
* v2 scripts: SDK contract enforced
```

# M27 Source Runtime SDK and Normalization Layer

## Goal

Improve the source runtime in staged lanes instead of rewriting the whole source system in one branch.

Each lane must be independently reviewable and mergeable. The roadmap moves from the safest boundary work first, then adds SDK helpers and display normalization only after typed failures and diagnostics redaction are stable.

## Hard Boundary

- No ReaderNext route semantics change.
- No ReaderNext render pipeline rebuild.
- No cache migration.
- No cookie migration.
- No appdata migration.
- No source install/update workflow rewrite.
- No source repository management UI rewrite.
- No network signing/auth redesign.
- No bypass logic for third-party source restrictions.
- No source identity or source key mutation.
- No OpenCC-converted text may replace source identity, DB keys, option values, callbacks, URLs, selectors, filenames, command keys, user input, or persisted payloads.
- Source scripts must not mutate global app settings directly.
- Source script contents must not be dumped into diagnostics by default.

## Runtime Responsibility Model

Application owns:

- storage
- UI
- settings
- diagnostics policy
- language preference
- source install/update policy
- cache policy
- source identity
- display-text normalization policy

Source Runtime SDK owns:

- safe capabilities exposed to scripts
- typed source result/error envelope
- common JSON/API validation helpers
- optional HTML parsing helpers
- diagnostics redaction boundary
- runtime API versioning
- capability validation

Source Script owns:

- source-specific URL construction
- source-specific payload mapping to common comic/chapter/page models
- source-specific selectors and parsing rules
- declared capabilities when migrated to SDK mode

## M27-L4 Display Text Normalization

### Scope

- Add Dart/app-layer `RemoteTextNormalizer`.
- Normalize source-provided display labels based on app locale and setting.
- Keep normalization render-time or adapter-time only.
- Do not persist converted labels.

### First Rollout Surfaces

- tag display labels
- category titles
- category item labels
- source button labels
- section titles
- JS dialog titles
- JS action button labels
- JS select option display labels

### Deferred Until Audit

- JS message body
- JS dialog content
- search/category option values
- source error message text
- comic titles
- filenames
- user input
- download names

### Hard Rule

Converted text is UI-only. Never convert or overwrite:

- providerKey
- namespace
- tagKey
- sourceKey
- sourceSha
- sourceVersion
- DB ID
- option value
- callback payload
- callback ID
- command key
- method name
- URL
- selector
- filename
- user input
- source object field used for identity/matching

### Suggested API

```dart
enum RemoteTextSurface {
  tagLabel,
  categoryLabel,
  commandLabel,
  sourceButtonLabel,
  menuLabel,
  sectionTitle,
  jsDialogTitle,
  jsActionLabel,
  jsSelectOptionLabel,
}

abstract final class RemoteTextNormalizer {
  static String normalizeLabel(
    String value, {
    required RemoteTextSurface surface,
    required Locale locale,
    bool? enabled,
    ChineseTextConverter? s2t,
    ChineseTextConverter? t2s,
  });
}

typedef ChineseTextConverter = String Function(String input);
```

Rules:

- `enabled == false` returns original value.
- `enabled == null` reads `appdata.settings[AppSettingKeys.enableRemoteChineseTextConversion.name] == true`.
- Non-zh locale returns original value.
- Traditional target: `languageCode == zh` and `scriptCode == Hant` or `countryCode == HK/TW`.
- Simplified target: `languageCode == zh` and `scriptCode == Hans` or `countryCode == CN/SG`.
- Traditional target calls existing OpenCC simplified-to-traditional wrapper.
- Simplified target calls existing OpenCC traditional-to-simplified wrapper.
- Empty, whitespace-only, ASCII-only, URI, file path, selector, or code-ish strings return original value.
- Do not log full label content.

Guard details:

- A single `/` inside a human label is not enough to treat it as a path.
- Guard only obvious paths:
  - starts with `/`
  - contains `://`
  - contains `\\`
  - matches drive path such as `C:\\`
  - contains `./` or `../`
  - has common file extension such as `.js`, `.json`, `.html`, `.jpg`, `.png`, `.webp`, `.zip`
- Guard code-ish strings containing `function(`, `=>`, `{`, `}`, `$.`, `document.`, or `querySelector`.

Diagnostics:

- Do not emit per-label converted trace by default.
- Emit `AppDiagnostics.warn('text.normalization', 'text.normalization.failed', ...)` only on conversion failure.
- Warn data: `surface`, `locale`, `direction`, `inputLength`, `source: remote_label`, `errorType`.
- Do not include raw label text in diagnostics.

## M27-L7 Optional HTTP Helper SDK

### Scope

- Add `runtime.http.request()` only after diagnostics redaction and typed error boundary are stable.
- Apply existing app network policy.
- Support timeout.
- Support response types:
  - `json`
  - `html`
  - `bytes`
  - `text`
- Redact sensitive request/response data before diagnostics.
- Return typed response or typed failure.

### Helper Contract

```js
runtime.http.request({
  method: "GET",
  url,
  headers,
  query,
  body,
  timeoutMs,
  responseType: "json",
  redact: ["authorization", "cookie", "x-auth-signature"],
});
```

Typed response:

```js
{
  ok: true,
  status: 200,
  headers: {},
  body: {},
  rawBodyPreview: '<redacted or limited>'
}
```

Typed failure:

```js
{
  ok: false,
  error: {
    code: 'SOURCE_UNAVAILABLE',
    message: 'Request failed',
    stage: 'loadInfo'
  }
}
```

### Out of Scope

- No automatic migration of existing source scripts.
- No cookie/auth redesign.
- No retry policy change unless explicitly specified.
- No bypass logic for third-party source restrictions.

### Acceptance Criteria

- Network failure -> `SOURCE_UNAVAILABLE`.
- Timeout -> `SOURCE_UNAVAILABLE` or `SOURCE_RATE_LIMITED` depending policy.
- Auth failure -> `SOURCE_AUTH_REQUIRED`.
- Request diagnostics are redacted.
- Existing source request path remains compatible unless a source explicitly opts into SDK HTTP helper.

## Merge Order

```text
1. M27-L1 Runtime Error Boundary
2. M27-L2 Source Diagnostics Redaction
3. M27-L3 JSON/API Helper and CopyManga Pilot
4. M27-L4 Display Text Normalization
5. M27-L5 HTML Helper SDK
6. M27-L6 Capability Declaration and SDK Versioning
7. M27-L7 Optional HTTP Helper SDK
```

Rules:

- L1 must merge first because later lanes depend on typed failure handling.
- L2 should merge before helper/API lanes because diagnostics must be safe before richer runtime metadata is added.
- L3 may start after L1 but should merge after L2 if it emits new diagnostics.
- L4 can be developed in parallel with L3, but should not merge before L1/L2.
- L7 is highest regression risk and must merge last.

## Overall Exit Criteria

- Source runtime has a typed error boundary.
- Existing sources still run through compatibility mode.
- CopyManga pilot no longer throws raw `ReferenceError` / `TypeError` on non-success payloads.
- `code != 200` source responses become typed source errors where helper is used.
- Missing required JSON fields become typed schema errors where helper is used.
- Dynamic settings shape mismatch becomes typed settings error.
- Persisted diagnostics redact auth headers, device identifiers, signatures, cookies, and tokens.
- Source script contents are not dumped by default.
- Display text normalization is UI-only and never mutates source identity.
- HTML and JSON helpers exist for opt-in SDK-mode sources.
- Capability declaration exists and legacy sources still run without declaration.
- Source UI shows visible typed error states instead of blank panels.
- No ReaderNext, cache, cookie, appdata, source install, or source repository semantics are changed.
