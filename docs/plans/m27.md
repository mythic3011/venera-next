# M27 Source Runtime SDK and Normalization Layer

## Goal

Improve the existing comic source runtime in staged lanes instead of rewriting the whole source system in one branch.

Each lane must be independently reviewable and mergeable. The roadmap moves from the safest boundary work first, then adds SDK helpers and display normalization only after typed failures and diagnostics redaction are stable.

## Existing Authority

The existing source runtime authority lives under:
/

```text
lib/features/sources/comic_source/runtime/
```

Existing tests live under:

```text
test/comic_source_runtime/
```

M27 must extend the existing runtime authority instead of creating a parallel runtime package.

Existing files to reconcile first:

```text
lib/features/sources/comic_source/runtime/source_runtime_error.dart
lib/features/sources/comic_source/runtime/source_runtime_codes.dart
```

Rules:

- Do not create a second `SourceRuntimeError` authority.
- Do not create a new `lib/features/source_runtime/` tree unless a later refactor explicitly moves the existing runtime.
- New runtime code should be placed under `lib/features/sources/comic_source/runtime/` or an existing adjacent source-runtime directory.
- New tests should be placed under `test/comic_source_runtime/` unless an existing test layout proves a better current location.

## Hard Boundary

- No ReaderNext route semantics change.
- No ReaderNext render pipeline rebuild.
- No cache migration.
- No cookie migration.
- No appdata migration.
- No source install/update workflow rewrite.
- No source repository management UI rewrite.
- No network signing/auth redesign.
- No bypass logic for third-party source restrictions.
- No source identity or source key mutation.
- No OpenCC-converted text may replace source identity, DB keys, option values, callbacks, URLs, selectors, filenames, command keys, user input, or persisted payloads.
- Source scripts must not mutate global app settings directly.
- Source script contents must not be dumped into diagnostics by default.

## Runtime Responsibility Model

Application owns:

- storage
- UI
- settings

Source Runtime SDK owns:

- common JSON/API validation helpers

- runtime API versioning
- capability validation

Source Script owns:

- source-specific URL construction
- source-specific payload mapping to common comic/chapter/page models
- source-specific selectors and parsing rules
- declared capabilities when migrated to SDK mode

## Existing Error Taxonomy Reconciliation

The repo already has `source_runtime_codes.dart`, which currently uses existing runtime code names such as `LEGACY_UNKNOWN` and `REQUEST_TIMEOUT`.

M27 must not replace this taxonomy blindly. L1 must add an explicit compatibility mapping from the existing codes to the proposed `SOURCE_*` names before any UI or tests depend on the new names.

Suggested mapping policy:

```text
LEGACY_UNKNOWN -> SOURCE_RUNTIME_EXCEPTION
REQUEST_TIMEOUT -> SOURCE_UNAVAILABLE
NETWORK_ERROR / REQUEST_FAILED -> SOURCE_UNAVAILABLE
AUTH_REQUIRED / LOGIN_REQUIRED -> SOURCE_AUTH_REQUIRED
FORBIDDEN -> SOURCE_FORBIDDEN
NOT_FOUND -> SOURCE_NOT_FOUND
SCHEMA_INVALID / INVALID_SHAPE -> SOURCE_SCHEMA_INVALID
SETTINGS_INVALID -> SOURCE_SETTINGS_INVALID
CAPABILITY_UNSUPPORTED -> SOURCE_CAPABILITY_UNSUPPORTED
RATE_LIMITED -> SOURCE_RATE_LIMITED
```

If an existing code name already carries the correct meaning, keep it as the stored/internal code and expose the `SOURCE_*` equivalent only at the UI/error-envelope boundary. Do not duplicate equivalent code constants without a compatibility reason.

Common target error meanings:

- `SOURCE_BLOCKED`
- `SOURCE_REQUIRES_OFFICIAL_APP`
- `SOURCE_AUTH_REQUIRED`
- `SOURCE_FORBIDDEN`
- `SOURCE_NOT_FOUND`
- `SOURCE_UNAVAILABLE`
- `SOURCE_SCHEMA_INVALID`
- `SOURCE_RATE_LIMITED`
- `SOURCE_SETTINGS_INVALID`
- `SOURCE_CAPABILITY_UNSUPPORTED`
- `SOURCE_RUNTIME_EXCEPTION`

## Shared Typed Result Model

The typed result model is introduced by Lane 1 and reused by later lanes.

Do not add a second `SourceRuntimeError` class if the existing `source_runtime_error.dart` already provides one. Extend the existing class or add a small wrapper/adaptor only if the current API cannot represent stage, user message, diagnostic message, and code.

Target shape:

```dart
sealed class SourceRuntimeResult<T> {}

class SourceRuntimeSuccess<T> extends SourceRuntimeResult<T> {
  SourceRuntimeSuccess(this.value);
  final T value;
}

class SourceRuntimeFailure<T> extends SourceRuntimeResult<T> {
  SourceRuntimeFailure(this.error);
  final SourceRuntimeError error;
}
```

The existing `SourceRuntimeError` authority should be able to represent:

```dart
class SourceRuntimeError {
  const SourceRuntimeError({
    required this.code,
    required this.stage,
    required this.userMessage,
    this.diagnosticMessage,
  });

  final String code;
  final String stage;
  final String userMessage;
  final String? diagnosticMessage;
}
```

## Shared Source Result Models

These models are the target contract for SDK-mode source outputs. Legacy sources may continue to be adapted through the compatibility wrapper.

```ts
type SourceComicInfo = {
  sourceKey: string;
  upstreamComicRefId: string;
  title: string;
  titleOriginal?: string;
  aliases?: string[];
  authors?: SourceTextValue[];
  tags?: SourceTag[];
  coverUrl?: string;
  description?: string;
  status?: string;
  language?: string;
  updatedAt?: string;
  groups?: SourceChapterGroup[];
};

type SourceChapter = {
  chapterRefId: string;
  title: string;
  titleOriginal?: string;
  index?: number;
  groupRefId?: string;
  publishedAt?: string;
  pageCount?: number;
};

type SourcePage = {
  pageRefId: string;
  imageUrl?: string;
  localPath?: string;
  width?: number;
  height?: number;
  headers?: Record<string, string>;
};

type SourceTag = {
  original: string;
  sourceKey?: string;
  searchFoldKey?: string;
};
```

Rules for tag/text fields:

- `original` preserves source-provided text.
- `sourceKey` is a stable provider/source key if available.
- `searchFoldKey` is an optional derived search key and must never replace source identity.
- Converted display labels are transient adapter/render output, not serialized source payload.
- OpenCC-converted text must not become provider identity or persisted source payload.

If a later implementation needs a display label field, use an adapter-only structure such as:

```ts
type DisplaySourceTag = SourceTag & {
  display: string;
};
```

This adapter-only display value must not be written back to source payload, DB rows, source maps, callback payloads, or cache entries that store source identity.

## Multi-Lane Roadmap

## M27-L1 Runtime Error Boundary

### Scope

- Extend the existing runtime under `lib/features/sources/comic_source/runtime/`.
- Reconcile `source_runtime_error.dart` and `source_runtime_codes.dart` with the target typed error envelope.
- Add Dart-side `SourceRuntimeResult` if no equivalent exists.
- Wrap legacy source calls:
  - `dynamicSettings`
  - `loadInfo`
  - `loadChapters`
  - `loadPages`
  - `search`
- Catch raw JavaScript exceptions and map them to typed errors.
- Attach source/stage context to failures.
- Preserve valid legacy output unchanged except for the typed wrapper envelope.
- Show a visible UI error state instead of blank panels when a source runtime failure occurs.

### Out of Scope

- No new HTTP helper.
- No OpenCC conversion.
- No tag normalization.
- No capability declaration.
- No source script rewrite except wrapper integration.
- No ReaderNext/cache/cookie/appdata/source install semantics change.

### Error Mapping

- `ReferenceError` -> `SOURCE_RUNTIME_EXCEPTION` equivalent.
- `TypeError` -> `SOURCE_RUNTIME_EXCEPTION` equivalent by default.
- Proven missing required payload field -> `SOURCE_SCHEMA_INVALID` equivalent.
- Invalid dynamic settings shape -> `SOURCE_SETTINGS_INVALID` equivalent.
- Unknown raw JS exception -> `SOURCE_RUNTIME_EXCEPTION` equivalent.

### Acceptance Criteria

- Raw `ReferenceError` does not reach UI.
- Raw `TypeError` does not reach UI.
- Invalid dynamic settings shape becomes a typed settings error.
- Existing runtime code taxonomy is reconciled through mapping or aliases.
- Valid legacy source output remains unchanged except for wrapper envelope.
- Source UI renders a visible typed error state.
- Failure object includes `sourceKey`, `stage`, and typed `code` or mapped equivalent.

## M27-L4 Display Text Normalization

### Scope

- Add Dart/app-layer `RemoteTextNormalizer`.
- Normalize source-provided display labels based on app locale and setting.
- Keep normalization render-time or adapter-time only.
- Do not persist converted labels.

### First Rollout Surfaces

- tag display labels
- category titles
- category item labels
- source button labels
- section titles
- JS dialog titles
- JS action button labels
- JS select option display labels

### Deferred Until Audit

- JS message body
- JS dialog content
- search/category option values
- source error message text
- comic titles
- filenames
- user input
- download names

### Hard Rule

Converted text is UI-only. Never convert or overwrite:

- providerKey
- namespace
- tagKey
- sourceKey
- sourceSha
- sourceVersion
- DB ID
- option value
- callback payload
- callback ID
- command key
- method name
- URL
- selector
- filename
- user input
- source object field used for identity/matching

### Suggested API

```dart
enum RemoteTextSurface {
  tagLabel,
  categoryLabel,
  commandLabel,
  sourceButtonLabel,
  menuLabel,
  sectionTitle,
  jsDialogTitle,
  jsActionLabel,
  jsSelectOptionLabel,
}

abstract final class RemoteTextNormalizer {
  static String normalizeLabel(
    String value, {
    required RemoteTextSurface surface,
    required Locale locale,
    bool? enabled,
    ChineseTextConverter? s2t,
    ChineseTextConverter? t2s,
  });
}

typedef ChineseTextConverter = String Function(String input);
```

Rules:

- `enabled == false` returns original value.
- `enabled == null` reads `appdata.settings[AppSettingKeys.enableRemoteChineseTextConversion.name] == true`.
- Non-zh locale returns original value.
- Traditional target: `languageCode == zh` and `scriptCode == Hant` or `countryCode == HK/TW`.
- Simplified target: `languageCode == zh` and `scriptCode == Hans` or `countryCode == CN/SG`.
- Traditional target calls existing OpenCC simplified-to-traditional wrapper.
- Simplified target calls existing OpenCC traditional-to-simplified wrapper.
- Empty, whitespace-only, ASCII-only, URI, file path, selector, or code-ish strings return original value.
- Do not log full label content.

Guard details:

- A single `/` inside a human label is not enough to treat it as a path.
- Guard only obvious paths:
  - starts with `/`
  - contains `://`
  - contains `\\`
  - matches drive path such as `C:\\`
  - contains `./` or `../`
  - has common file extension such as `.js`, `.json`, `.html`, `.jpg`, `.png`, `.webp`, `.zip`
- Guard code-ish strings containing `function(`, `=>`, `{`, `}`, `$.`, `document.`, or `querySelector`.

Diagnostics:

- Do not emit per-label converted trace by default.
- Emit `AppDiagnostics.warn('text.normalization', 'text.normalization.failed', ...)` only on conversion failure.
- Warn data: `surface`, `locale`, `direction`, `inputLength`, `source: remote_label`, `errorType`.
- Do not include raw label text in diagnostics.

- Modify: `lib/pages/settings/settings_schema.dart`
- Modify: `lib/foundation/appdata.dart`
- Modify: `lib/pages/settings/app_settings_strings.dart`

### Acceptance Criteria

- zh_HK / zh_TW users see Traditional remote tag/category/JS display labels when setting is enabled.

## Merge Order

```text
1. M27-L1 Runtime Error Boundary
2. M27-L2 Source Diagnostics Redaction
3. M27-L3 JSON/API Helper and CopyManga Pilot
4. M27-L4 Display Text Normalization
5. M27-L5 HTML Helper SDK
6. M27-L6 Capability Declaration and SDK Versioning
7. M27-L7 Optional HTTP Helper SDK
```

Rules:

- L1 must merge first because later lanes depend on typed failure handling.
- L2 should merge before helper/API lanes because diagnostics must be safe before richer runtime metadata is added.
- L3 may start after L1 but should merge after L2 if it emits new diagnostics.
- L4 can be developed in parallel with L3, but should not merge before L1/L2.
- L7 is highest regression risk and must merge last.

## Overall Exit Criteria

- Source runtime has a typed error boundary.
- Existing sources still run through compatibility mode.
- CopyManga pilot no longer throws raw `ReferenceError` / `TypeError` on non-success payloads.
- `code != 200` source responses become typed source errors where helper is used.
- Missing required JSON fields become typed schema errors where helper is used.
- Dynamic settings shape mismatch becomes typed settings error.
- Existing source runtime code taxonomy is reconciled with the target source error meanings.
- Persisted diagnostics redact auth headers, device identifiers, signatures, cookies, and tokens.
- Source script contents are not dumped by default.
- Display text normalization is UI-only and never mutates source identity.
- HTML and JSON helpers exist for opt-in SDK-mode sources.
- Capability declaration exists and legacy sources still run without declaration.
- Source UI shows visible typed error states instead of blank panels.
- No ReaderNext, cache, cookie, appdata, source install, or source repository semantics are changed.
